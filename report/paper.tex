\begin{abstract}
We adapt AlphaGateau from chess to the abstract strategy game Breakthrough using a board-size-invariant graph encoder, and an AlphaZero-style self-play training loop.
We compare a graph neural network (GNN) against a convolutional baseline (CNN) on 8$\times$8
Breakthrough over 40 self-play iterations, and also study knowledge transfer from 5$\times$5
pretraining to 8$\times$8 fine-tuning.
The CNN converges to 100\% greedy-evaluation win rate within 40 iterations while the GNN
plateaus at 42\% with high variance throughout training.
Transfer from 5$\times$5 pretraining yields a strong early benefit (67\% win rate at iteration 5
versus 17\% from scratch) but does not close the long-run gap with the CNN baseline.

\section{Introduction}


Breakthrough is a two-player abstract strategy game played on an $8\times8$ board in which each side controls only pawns.
Despite its simple rules, the game is strategically rich and has a large state space of approximately $1.5\times 10^{16}$ \cite{jonsson2025solving}.
Figure~\ref{fig:rules} illustrates the starting position and the legal forward and diagonal-capture moves that define the game.

Breakthrough is a useful testbed for studying learned representations in reinforcement learning.
Its rules are simple, but successful play depends heavily on local geometry: pieces move only forward, captures are diagonal, and reaching the opponent's back rank ends the game.
This structure makes the game a natural candidate for graph-based representations, in which squares are nodes and legal relations are edges rather than purely image-like grid encodings.
It is especially attractive for transfer experiments because the same local move rules apply across board sizes.

Our project studies two questions.
First, under a fixed self-play training budget, does an AlphaGateau-style graph neural network (GNN) outperform a simpler convolutional neural network (CNN) baseline on 8$\times$8 Breakthrough?
Second, does the GNN's board-size-invariant representation make transfer from 5$\times$5 pretraining to 8$\times$8 fine-tuning meaningful?
To answer these questions, we built a JAX-based self-play training and evaluation pipeline for Breakthrough and compared graph and convolutional agents under matched settings.
Our main findings are that the CNN baseline outperforms the GNN from scratch on 8$\times$8, while 5$\times$5 pretraining gives the GNN a clear early advantage during transfer without closing the full gap to the
CNN.

\section{Background}

AlphaZero-style agents combine self-play reinforcement learning with Monte Carlo Tree Search
(MCTS)~\cite{alphazero}.
A neural network predicts a policy over actions and a scalar value estimate; MCTS then refines
action selection during play, and the search distribution becomes a policy target for training.
We use Gumbel MCTS\footnote{using the \texttt{mctx} library}, which selects actions by
differentiably perturbing the policy logits with Gumbel noise~\cite{muzero}.

AlphaGateau~\cite{alphagateau} replaces the convolutional encoder in AlphaZero with a graph
neural network, aiming to better represent the relational structure of board games.
It uses graph attention~\cite{gat} with residual message passing layers.

Prior work on Breakthrough has focused primarily on exact game solving rather than self-play learning.
Early work combined race patterns with job-level proof-number search to solve the 6$\times$5 variant as a second-player win~\cite{saffidine2011breakthrough}.
More recently, the 6$\times$6 game was weakly solved as a first-player win using a hybrid of endgame tablebases,
race patterns, proof-number-based search, and a neural-network-guided job scheduler~\cite{jonsson2025solving}.
These results show that, despite simple rules, Breakthrough is strategically rich enough to support substantial search and solving research.

Those exact-solving systems aim to prove the game-theoretic value of positions.
Our setting is different: we do not attempt to solve Breakthrough exactly, but instead study whether learned policy/value models can play effectively under self-play.
In particular, we care about representation quality and cross-board transfer rather than formal game-theory optimality.

Breakthrough~\cite{breakthrough} is especially attractive for this representation question because the same local move rules apply on both 5$\times$5 and 8$\times$8 boards.
A fixed-size CNN processes an 8$\times$8 grid directly but does not naturally reuse the same architecture across board sizes.
By contrast, a graph encoder can preserve the same feature schema while changing only the number of nodes and edges.
This makes Breakthrough a clean testbed for our two questions: whether a GNN is competitive with a CNN under the same RL budget, and whether a board-size-invariant graph representation makes 5$\times$5~$\rightarrow$~8$\times$8 transfer meaningful.



\section{Methodology}

\paragraph{Environment and action encoding.}
We implemented a custom Breakthrough envronment in JAX.
States include the board array, an observation tensor, a legal-action mask, the side to move, rewards, and terminal game state.
Actions use a fixed encoding $\texttt{action} = \texttt{from\_square} \times 3 + \texttt{move\_type}$,
where \texttt{move\_type} $\in$ \{\texttt{forward}, \texttt{diag\_left}, \texttt{diag\_right}\}.
We canonicalize the board so the player to move always advances upward, keeping both the tensor
and graph interfaces board-size agnostic.

\paragraph{Model representations.}
The CNN baseline uses a three-channel tensor (self pieces, opponent pieces, empty squares) fed
into a residual convolutional network.
The GNN constructs a graph with one node per square and directed move edges for both players.\footnote{This is in line with the AlphaGateau paper.}
Node features encode occupancy, normalized coordinates, and edge-of-board indicators; edge features
encode move direction, owner, legality, capture structure, and goal-rank information.
This graph construction preserves the same local feature schema across board sizes, which is the
basis for our transfer experiments.

\paragraph{Learning architectures and training procedure.}
For the AlphaZero-style agents, both the CNN and GNN use hidden size 64 and four residual layers.
The GNN adds attention pooling over node embeddings to produce a global representation.
Self-play training uses 32 MCTS simulations per move, a 50k-sample replay buffer, and
32-sample minibatches.
Unless otherwise noted, the learning rate is $10^{-3}$.

\paragraph{Experiment configurations.}
We ran four groups of experiments.
First, in the 8$\times$8 scratch comparison, the CNN and GNN were trained from scratch for
40 iterations\footnote{Fourty iterations were used for reason X, Y, Z.} %TODO outline why 40
with 64 self-play games per iteration and a 192-ply move cap.
Second, in the transfer setup, we pretrained the GNN on 5$\times$5 for 40 iterations using
128 self-play games per iteration, then fine-tuned it on 8$\times$8 for 30 iterations.\footnote{Thirty iterations were used for finetuning for reason X,Y,Z.} %TODO outline why 30.
Third, we repeated selected GNN runs with cosine decay in place of a fixed learning rate to study
whether a learning-rate schedule improved optimization.
Fourth, we trained a TD($\lambda$) baseline using a value-only CNN with no policy head or MCTS.
For TD($\lambda$), we use $\lambda = 0.7$ and $\epsilon$-greedy self-play; the 5$\times$5 run
uses 128 episodes per iteration and a 96-ply move cap, while the 8$\times$8 run uses 64 episodes
per iteration and a 256-ply move cap, with both running for 40 iterations at learning rate
$10^{-3}$.
We also explored an extended transfer pipeline of 5$\times$5 to 6$\times$6 to 8$\times$8 as an
additional experiment aimed at bridging the gap between board sizes.

\paragraph{Evaluation protocol.}
Every five iterations we play 12 games against a greedy evaluator and record the win rate.
For the policy-based agents, the greedy evaluator always picks the highest-policy action.
After training, we run a 12-game head-to-head match between the final GNN and CNN scratch
checkpoints.
We also record zero-shot 8$\times$8 performance of the 5$\times$5 pretrained model before any
fine-tuning.
Since our TD baseline has no policy head, it is evaluated using greedy value-based play, selecting
the action whose successor state has the highest predicted value rather than using MCTS.

\paragraph{Scratch comparison.}
The CNN and GNN both start near zero win rate but diverge sharply from iteration 15 onward.
The CNN climbs monotonically, reaching 0.58 at iteration 15, 0.92 at iteration 25, and 1.00 at
iteration 40.
The GNN peaks at 0.67 at iteration 20, then oscillates between 0.25 and 0.58 for the rest of
training, ending at 0.42.
The policy loss curves confirm this: the CNN's policy loss descends steadily to 1.97 by iteration
40, while the GNN's policy loss plateaus at approximately 2.05 from iteration 36 onward, indicating
convergence to a suboptimal policy.

The head-to-head match between final checkpoints confirms the scratch training advantage: the CNN
wins 10 of 12 games (83\%), with the GNN winning only 2 (17\%).

\paragraph{Transfer learning.}
The 5$\times$5 pretrained GNN converges cleanly, reaching 1.00 win rate by iteration 20 and
maintaining it through iteration 40.
Applied zero-shot to the 8$\times$8 board, it achieves only 17\% win rate (2/12 games), showing
that the 5$\times$5 policy does not transfer directly to the larger board.

Fine-tuning from the 5$\times$5 checkpoint, however, gives a substantial early-stage advantage
(Figure~\ref{fig:transfer}, right panel).
At iteration 5, the fine-tuned model wins 67\% of games versus a greedy evaluator, compared to
only 17\% for the GNN trained from scratch at the same iteration---a 4$\times$ improvement.
This demonstrates that the board-size-invariant GNN representation does enable meaningful
cross-board knowledge transfer.
By iteration 30, the fine-tuned model reaches 67\% while the scratch GNN at the same iteration
is at 58\%, a persistent but modest advantage.
Neither GNN variant approaches the CNN's 75\% win rate at iteration 30.

\paragraph{TD($\lambda$) baseline.}
TD($\lambda$) really struggles against the greedy evaluator: it is never able to win a game at any iteration of training and on either board size. On the 5$\times$5 board, we see a short spike at iteration 2 caused by the network outputting its first non-zero values. The loss then decreases from 0.186 at iter 3 to 0.100 at iter 8. After that, loss slowly deteriorates to 0.146 at iter 40, with the number of transitions decreasing suggesting the model is converging to a fast-losing policy. On the 8$\times$8 board, we see a similar spike at iter 2, but the loss never improves from there. This indicates the model does not learn any meaningful information or favor any winning moves.

\paragraph{Cosine decay LR.}
The cosine decay learning rate scheduler allows the GNN to reach 100\% win rate twice as fast as the fixed LR on the 5$\times$5 board. However, we see that the learning rate drops by the final iteration to 83\%. We attribute this to evaluation variance: the evaluator "got lucky". Neither the policy nor value losses showed any degradation corresponding to the win rate drop.
On the 8$\times$8 board, we note that we obtain a lower peak win rate than fixed LR, but a higher final win rate. We also find that while the loss plateaued for the fixed LR, it continues to steadily decrease with cosine decay. This alongside the noisy win rates indicates that the model has not yet converged. We posit that with more iterations, this model would overcome the fixed LR's peak win rate.
The 5$\times$5 to 8$\times$8 transfer learning pipeline also shows clear improvement: the win rate jumps from 0.75 and 0.67 (peak and final) for fixed LR to 0.83 (both) for cosine decay. The win rate is also much less noisy for cosine decay, and consistently improves: 75\% at iters 5 and 10, 83\% at iters 15, 20, 25, 30.

\section{Discussion}

\paragraph{Systems outcome.}
The project delivers a complete, reproducible AlphaZero-style Breakthrough stack: custom JAX
environment, graph and convolutional models, resumable self-play training, automated report-asset
generation, and Slurm-native wrappers for the course cluster.
The graph encoder is genuinely board-size invariant, and the pipeline demonstrates the
5$\times$5~$\rightarrow$~8$\times$8 transfer question end-to-end.

\paragraph{Why the CNN outperforms the GNN.}
The CNN's advantage on 8$\times$8 Breakthrough is a meaningful negative result.
One likely explanation is inductive bias: on a regular grid, convolutions share weights across
all spatial positions and efficiently capture the local, directional patterns that dominate
Breakthrough tactics.
The GNN introduces additional parameters through attention pooling and edge feature MLPs, making
it harder to optimize under a fixed 40-iteration budget with a single learning rate.
The GNN policy loss plateau at $\approx$2.05 suggests the model converged to a saddle point or
local minimum.
We hypothesize that larger iteration budget, layer-wise or decaying learning rates, or a simplified GNN variant
without attention pooling might close this gap. The cosine decay experiments confirm this assumption: the GNN's policy loss is still decreasing at iteration 40, avoiding the plateau, and the final win rate improves.

\paragraph{Transfer benefit and limits.}
Transfer from 5$\times$5 pretraining produces a real, quantifiable early-training benefit:
the fine-tuned model starts 4$\times$ higher than the scratch GNN at iteration 5.
This confirms our second hypothesis: the GNN's board-size-invariant representation supports
meaningful knowledge transfer.
However, the benefit is largest early in fine-tuning and shrinks as both models train longer.
This suggests the pretrained features provide a good initialisation rather than a structural
advantage that compounds.
An interesting follow-up is whether increasing the fine-tuning budget beyond 30 iterations would
allow the fine-tuned GNN to match or surpass the CNN.

\paragraph{Evaluation noise.}
Each win-rate data point comes from 12 games, giving a standard deviation of approximately
$\sqrt{p(1-p)/12} \approx 0.14$ for $p \approx 0.5$.
The oscillations in the GNN win-rate curve---most visible between iterations 20 and 40---are
consistent with this level of noise around a slowly moving mean.
A larger evaluation budget (e.g., 50--100 games per checkpoint) would substantially reduce this
variance in future runs.

\paragraph{TD($\lambda$) baseline.}
TD($\lambda$) was never able to beat its evaluator. We attribute this to the evaluator being too strong: it is given an explicit lookahead that TD is not able to rival within 40 iterations. The other models make use of MCTS which provides a one-step lookahead, but TD's value function is not strong enough to approximate that in the same number of iterations. This demonstrates the need for the AlphaZero approach.

\paragraph{Learning Rate Schedule.}
We have demonstrated that implementing a cosine decay learning rate scheduler significantly improves performance on GNN models. We speed up the learning, avoid plateaus, and reduce win rate variability. %TODO: rephrase variability?

\section{Conclusion and Future Work}

We successfully adapted AlphaGateau to Breakthrough, built a complete reproducible pipeline, and
ran all four planned experiments to completion.
The main empirical findings are: (1) the CNN baseline outperforms the GNN from scratch under the
current 40-iteration budget, with the GNN's policy loss plateauing at a higher value; (2)
transfer from 5$\times$5 pretraining meaningfully accelerates early 8$\times$8 learning but does
not fully compensate for the GNN's convergence disadvantage versus the CNN; (3) the TD learning fails to rival a greedy evaluator, demonstrating the need for MCTS lookahead; and (4) cosine LR scheduling considerably improves speed and performance.

Promising future directions include: running a longer fine-tuning budget (60--80 iterations) to
test whether the fine-tuned GNN can eventually match the CNN; running ablations that disable attention pooling or simplify edge features; increasing the evaluation game count to reduce win-rate noise; and repeating with the updated 256-ply cap
to eliminate any truncation artefacts from the current 192-ply limit.
